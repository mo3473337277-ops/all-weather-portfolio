## 二、核心逻辑详解

### 2.1 数据加载（`data.py`）

**7 资产面板加载流程**:

| 资产 | 加载函数 | 数据来源 | 特殊处理 |
|------|----------|----------|---------|
| hs300 | `_load_with_index()` | ETF(510300) + hs300_idx | 指数拼接 |
| us_sp500 | `_load_sp500_cny()` | ETF(513500) + sp500_idx×USDCNY | 汇率换算 |
| credit | `_load_with_index()` | ETF(511220) + credit_idx | 指数拼接 |
| bond_10y | `_load_bond_10y()` | ETF(511260) + treasury_idx + credit_idx | 三级回退 |
| bond_30y | `synthesize_bond_30y()` | 三阶段合成 | 久期放大/利差法/ETF |
| gold | `_load_gold_cny()` | ETF(518880) + 伦敦金×USDCNY + 持仓推算 | 三级回退+汇率 |
| nonferr | 手动拼接 | ETF(159980) + nonferr_idx + shfe_copper | 三级回退 |
| wti | `_load_wti_cny()` | SC原油 SC0 (2018+) + WTI USD×USDCNY proxy | 拼接，无QDII限制 |
| copper | `_load_copper()` | 沪铜 CU0 (2005+) | 展期成本扣减0.3%/年 |

**30Y 国债合成三阶段**:
1. 2015-01 ~ 2020-02：×3.0 久期放大（扣减 0.3%/年）
2. 2020-02 ~ 2024-03：利差法（10Y-30Y spread × duration 18.0）
3. 2024-03 ~ now：ETF 511130 真实 NAV

**安全扣减**:
- `nonferr`: 0.5%/年（申万有色指数不含管理费）
- `bond_30y`: 0.3%/年（久期放大期权费率差）
- `wti`: 1.0%/年（QDII 基金管理费）

### 2.2 回测引擎（`backtest.py`）

**统一入口 `backtest()`** 支持两种加权方式:

| 参数 | V3c 多元 | V3-B 风险平价 | V3-B 保守增强 |
|------|----------|--------------|--------------|
| weighting_method | inverse_vol | hierarchical_rp | inverse_vol |
| 回看窗口 | 60d | 20d | 20d |
| max_w | 0.30 | 0.20 | 0.25 |
| min_w | 0.03 | 0.02 | 0.02 |
| 桶 | 无 | 4 桶等权 | 无 |

**主循环热路径**（~5300 日 × 3 策略）:

```
每日:
  1. h * rets_arr[i] → 组合日收益（numpy dot）
  2. h *= (1 + rets_arr[i])，h /= sum(h)（权重漂移修正）
  3. 更新 gold/hs300 峰值
  
每月（调仓日）:
  4. 计算 trailing 窗口权重（inverse_vol 或 HRP）
  5. 计算各 SMA（nonferr 75d, gold 75d, sp500 120d）
  6. HS300 AND 抄底检查（PB 分位 + PE 分位 + 价格回撤 + SMA120）
  7. _apply_trend_dip(): 趋势过滤 + 抄底 boost + max_w clip
```

**趋势过滤优先级**:
1. nonferr < SMA75 → 清仓转移至 credit
2. us_sp500 < SMA120 → 清仓转移至 credit
3. gold < SMA75 → 清仓转移至 credit
4. gold 回撤 > 15% → 抄底 boost 2.5x（从 credit 扣）
5. HS300 AND 抄底（PB < 30%ile + 回撤 > 25% + 价格 > SMA120）→ boost 1.8x

### 2.3 风控模块（`risk.py`）

| 函数 | 作用 | 算法 |

| `hierarchical_rp_weights()` | 分层风险平价 | 桶内逆波动率 → 桶间等权/逆波动率 |
| `_precompute_percentile()` | PB/PE 分位预计算 | O(n²) 向量化 broadcast |

| `dynamic_cash_ratio()` | 动态现金比例 | 回撤 >20% → 0%，<5% → 30% |

### 细节说明

#### 1. D_excess n_sim 缩减（最高性价比）

`d_significance()` 在收益正态零假设下生成 10000 条模拟路径，每次分配 (10000, 5300) float64 ≈ 424MB。缩减到 2000 次后分位点估计精度约 0.7pp（原 0.3pp），对 5% 显著性判断完全够用。

#### 2. Bootstrap 向量化（最大单点收益）

当前实现用 Python for 循环跑 4000 次迭代（4 策略×1000），每次迭代创建 ~60 个 np.arange 数组再 concat。向量化后用一次 `rng.integers()` 生成全部起始索引，broadcast 构建索引矩阵，预测加速 8-20x。

#### 3. SMA 预计算

调仓日 SMA 计算用 `price_arr[max(0,i-window):i].mean()` 每次从头算起，相邻调仓日间隔 21 天但窗口 75-120 天，75-100 天数据完全重叠。预计算滚动均值后查表 O(1)。

#### 4. 并行回测（最大架构优化潜力）

三个策略各自有独立 numpy 数组，无共享可变状态，理论上可用 `ThreadPoolExecutor` 并行（numpy release GIL）。预计 step_2 从 25s → 10s。

## 四、已知设计决策

### 4.1 已否决方案

- **方案C 70/30 动态现金补仓**: 不敌纯 risk_parity 100%，补仓=摊平亏损（memory: strategy-c-negative-result）
- **V3-B 桶优化**: 前 2 次优化推荐均不可行（memory: v3b-bucket-optimization-failed）
- **方案B 风险平价桶(20d)**: 已移除，Sharpe 1.66，max/min 3.85x（memory: v3b-risk-parity-buckets）

### 4.2 关键设计约束

- **Bootstrap 代理权重**: 动态权重用最后调仓窗口固定权重近似（pipeline.py:215-216）
- **WTI 原油**: 已集成至数据管道和引擎，QDII 限购暂不可执行
- **Sharpe 修正**: (CAGR - 2.2%) / vol，原始版保留为 sharpe_raw
- **div_idx / soymeal**: 已于 2026-05-27 移除

### 4.3 数据备份策略

数据有 3 层备份：本地 CSV → `data/` → GitHub 仓库，任何一层丢失可从另两层恢复。
